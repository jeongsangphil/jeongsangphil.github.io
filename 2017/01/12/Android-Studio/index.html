resource

drawable : 바꿀수있는 그림
layout : 각 보여지는 페이지(layout) 설정, 새로추가 : layout resource file
종류 : FrameLayout(한페이지만), linearLayout(선으로 나눠짐), gravityLayout(정렬), TableLayout(순서상관x html table)

menu : 메뉴버튼누르거나 … 누르면 나오는 메뉴의 속성 (settings)
mipmap : 바꾸지 않고 기본으로 하나만 사용하는 그림 넣는곳
values : 값 변경

단축키

ctrl + q : 라이브러리, 사용법, 무슨값을넣어야하는지 보기
alt + enter : 오류 해결(넣어야할거 자동 추가)
ctrl + space : 넣을수있는경로나 값 추천
ctrl + alt + l : 자동정렬 (모두선택후)

log

log의 종류 : error/warn/debug/verbose
log.e(String tag, String msg); // tag는 activity/class 이름, msg는 표시할 메세지
log 갯수는 한정, 적당히 표시해야함

Thread

Main Thread (aka UI thread) : 모든 input / output
Background Thread : input / output 을 제외하고 나머지 모두 (기다려야하는것)

AsyncTask : onPreExcute, doInBackground (Background 스레드), onProgressUpdate, onPostExcute
syncAdapter : GOM이 알아서 업데이트해줌 (asyncTask랑은 반대)

함수와 비슷하다고 생각

function sum(num1,num2){}

sum(1,2);

위와같은 함수, 호출이있을때 num1과 num2가 파라미터(고정), 호출시 1과 2가 전달자라하면

Seoul:kr , json , metric , 7 , ebd4c9e7404d1c92b00c0f6638268b6a 는 전달자이다 (원하는대로 변경 가능)

Json

{} 대괄호로 시작되는 코드.

구조는 key와 value의 형태.

자료형 (value 형태) : Object형, 일반형 (대괄호로 시작, 1~여러개의 object), 배열형(중괄호[] 시작, 대괄호로 순서 구분)

화면 누르기

방법 ? : touch : 전체 / select (listview) / click

Toast

간단하게 표시하는 msg. ex) true / false같이 잠깐 표시(화면전환 x)

Activity

Activity : 화면 하나 전체

ActivityFragment : 화면에 표시할(갖다붙이는) 것들의 그룹, 화면 바꾸는 것.

StartActivity(Intent) : 화면 전환 (main Activity -> detail Activity)

Intent

Intent : Activity간 화면전환할 때 통신을 위한 방법. ex) db 로그인할때 값 전달, session

Explicit Intent(유기명) : 실제 이름 만들고 보내는사람 받는사람 존재

Implicit Intent(무기명) : 해야할 일은 정해져있으나 어떤 대상에 제한되지 않음

ex) 어플에서 전화번호 누르면 전화앱 여러개 선택할수있음, sms, 메일주소 등
