Implicit Intent(무기명) : 해야할 일은 정해져있으나 어떤 대상에 제한되지 않음

ex) 어플에서 전화번호 누르면 전화앱 여러개 선택할수있음, sms, 메일주소 등

사용방법

Explicit Intent(유기명)

Intent 선언후

  • i.putExtra(key, value) : 값 넣기
  • i.getStringExtra(name) : 값 받아오기 name은 key값
  • i.hasExtra(name) : 가지고있는지?
  • i.isEmpty ? : 비어있는지?

Implicit Intent(무기명)

Intent 선언후 , Action(행동)만 넣기, 수신자/발신자 없이

ACTION_SEND / ACTION_VIEW : 지도누르면 보여주거나 / 보내는 것 (해야할 일, action)

ex) 액션뷰로 geo를 보내면 지도앱들이 반응을하는 broadcast(메세지 전송) 이 일어남

share action provider : 무기명 만들기(공유버튼) , 데이터 전송시 provider형태로 데이터 전송(공유)하는 것

content provider

각각 앱의 고유한 저장공간을 연결해서 데이터 주고 받는 즉, 데이터 접근에대한 관리를 담당한다. (데이터 제공자)

접근을위해 db의 전체 contract를 알려줄 필요가 없이 요청하는 정보(필요한 정보)만 공유해주면된다.

또한 실제로 SQLite를 안쓰고 content provider로 돌아가는 것 처럼 보인다 (보안목적으로 좋음, 실제로는 SQLite로 돌아감)

ex) 어플 위젯(widget)

데이터를 다른앱에서도 효율적으로 이용(공유)하려면 permission 필요.

-> shared content provider사용 ex) 전화부, 문자 등의 데이터 공유

content provider외에도 content observer / content reserver가 있다.

content reserver : provider를 도와주는 역할 ex) dbhelper ..

content observer : 관찰하는 역할 ex) addAll() 함수 원형의 notification

데이터입력은 contentvalues를 이용해서 하면 된다.

resolver를 사용해서 observer에게 보여준다. (하위말고 상위에게)

table : 보통 테이블 이름 ex) weather

Query : 실제 원하는 쿼리 데이터 ex) seoul, kr

Custom View

보통 뷰를 상속해서 사용, 캔버스 방식

onMeasure : 자식뷰에서 부모뷰의 크기에 맞게 측정(설정)

​ ex) 100 x 100 이미지가 있을 때 부모 뷰의 크기가 100 x 90 이면 부모 뷰의 크기에 맞게 설정

onDraw : onMeasure에서 측정한 뷰를 직접 그리는 함수 (xml 수정하는것 아님 ?)

숙제 : 가로/세로 모드 디테일뷰에 방향 그림 출력 (원형 그림)
